1. Brief History of Computers Four Generations

A computer evolved from large, single-purpose calculating machines into the small, general-purpose devices we use today. Historians divide this evolution into four generations, each marked by a change in the core electronic component used to build the machine.

GenerationPeriodCore TechnologyKey Trait
First1940s–1950sVacuum tubesHuge, hot, unreliable, used machine language
Second1950s–1960sTransistorsSmaller, faster, less heat, assembly language
Third1960s–1970sIntegrated Circuits (ICs)Multiple transistors on a chip, high-level languages
Fourth1970s–presentMicroprocessorsEntire CPU on one chip; birth of the personal computer
Flowchart · Four Generations
Each generation solved the previous one's size, heat, or speed problem.
Examiner's Favourite "Discuss the four generations of computers" is one of the most frequently repeated long questions. Always pair each generation with its core technology AND one limitation it solved from the previous generation.

2. Elements of a Computer System

Every computer, regardless of size, is built from four functional elements that work together in a continuous cycle: Input → Processing → Output, supported by Storage.

  • Input: Devices and methods used to feed raw data into the computer (e.g. keyboard, mouse, scanner).
  • Processor (CPU): The "brain" of the computer; made of the Control Unit (directs operations), the Arithmetic & Logic Unit — ALU (performs calculations and comparisons), and Registers (tiny, ultra-fast storage inside the CPU).
  • Memory: Primary memory (RAM/ROM) holds data and instructions the CPU is currently working on; secondary memory stores data permanently.
  • Output: Devices that present processed data to the user (e.g. monitor, printer).

3. Hardware vs Software

Hardware is every physical, touchable part of a computer system — the CPU, monitor, keyboard, mouse, printer, circuit boards, and storage devices. Hardware cannot function meaningfully without instructions.

Software is the set of instructions (programs) that tells the hardware what to do. It is intangible — you cannot touch software, only its storage medium.

AspectHardwareSoftware
NaturePhysical, tangibleLogical, intangible
DamageWears out / can break physicallyDoes not wear out; can develop "bugs"
ExampleCPU, RAM, printerOperating system, MS Word

4. Application Software vs System Software

System Software

Manages and controls the computer's own hardware and resources so that application software can run. Example: Operating Systems (Windows, Linux), device drivers, utility programs.

Application Software

Designed for the end-user to perform specific tasks — word processing, spreadsheets, browsing, gaming.

Uses of Application Software

  • Automates routine office tasks (typing, calculations, presentations).
  • Supports communication, research, and data analysis.
  • Provides entertainment and creative tools.

Limitations of Application Software

  • Depends entirely on compatible system software/hardware to run.
  • General-purpose packages may not fit highly specialised needs.
  • Requires updates and can carry security vulnerabilities.

5. Importance & Limitations of Computers

Computers are valued for speed (millions of operations per second), accuracy (near-zero error when correctly programmed), storage capacity, and the ability to perform repetitive tasks without fatigue. They have transformed education, business, communication, medicine, and research.

Their main limitation is that they have no judgement or common sense — they only do exactly what they are instructed to do (the "Garbage In, Garbage Out" principle), and they depend completely on electricity and correct programming.

7. Computer Based Information System (CBIS)

A CBIS uses computer technology to carry out some or all of an organisation's information processing. It is more than just hardware and software — it is a complete system of five components working together to convert data into useful information.

  • Hardware — physical devices that process and store data.
  • Software — programs that direct the hardware.
  • Data — raw facts that the system processes into information.
  • People — users and IT staff who operate and manage the system.

All five components must work together — remove one, and the system fails in practice.

8. Methods of Input & Processing

Organisations choose how data enters a system and when it is processed:

  • Keyboard entry — manual typing; flexible but slower and error-prone.
  • Source Data Automation (SDA) — data is captured directly from its origin using devices like barcode readers or OCR, reducing transcription errors.
  • Batch Processing — data is collected over a period and processed together at a scheduled time (e.g. monthly payroll).
  • Online / Real-time Processing — data is processed immediately as it is entered (e.g. ATM withdrawal, airline booking).

9. Organising the Computing Facility

How an organisation arranges its computers and processing power falls into three broad models:

ModelDescriptionTrade-off
CentralizedAll processing power and data are kept on one large central computer; users connect via terminals.Easy to control & secure, but a single point of failure and can bottleneck under heavy load.
DistributedProcessing is spread across multiple connected computers that cooperate and share data, often across locations.Flexible and resilient, but harder to coordinate and secure.
DecentralizedEach department/location has its own independent computer with little or no central coordination.Maximum local control, but duplicated effort and inconsistent data.

10. Input Devices — Overview

Input devices are the gateway through which raw data enters a computer system. They range from manual devices like the keyboard to automated capture tools used in Source Data Automation. The detailed types — keyboards, terminals, pointing devices, voice input, and SDA technologies — are covered fully in Chapter 2: I/O Devices, Storage & Data Communication, since they connect directly to how data is processed and stored.
